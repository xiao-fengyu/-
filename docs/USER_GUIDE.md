## 1. 产品介绍

e-platform 是一款 **AI 驱动的商品图片生成 + 多平台一键上架** 桌面客户端。

### 核心功能

- 🎨 **AI 图片生成** — 连接 DALL-E 3、通义万相等提供商，根据商品描述自动生成电商图片
- 📦 **多平台发布** — 支持拼多多等电商平台，一键发布商品
- 📋 **批量模式** — 导入 Excel/CSV，批量生成图片 + 批量发布
- 🔄 **完整工作流** — 生成 → 确认/编辑 → 发布 → 日志追踪
- 💾 **本地数据** — SQLite 存储，支持备份/导出

### 技术架构

```
┌─────────────────────────────────────────────┐
│              Electron 桌面应用               │
├──────────────┬──────────────┬───────────────┤
│  React 前端   │ Express 后端  │   SQLite DB   │
│  (Ant Design) │  (REST API)  │  (本地存储)    │
├──────────────┴──────────────┴───────────────┤
│          AI 提供商    │    电商平台 API       │
└─────────────────────────────────────────────┘
```

## 2. 安装指南

### 方式一：Windows 安装包（推荐）

1. 从发布页下载 `e-platform-Setup-x.x.x.exe`
2. 双击运行安装程序
3. 选择安装目录（默认 `C:\Program Files\e-platform`）
4. 勾选"创建桌面快捷方式"
5. 完成安装，双击桌面图标启动

### 方式二：开发模式运行

```bash
# 1. 克隆仓库
git clone https://github.com/xiao-fengyu/- e-platform
cd e-platform

# 2. 安装依赖
npm install

# 3. 启动开发模式
npm run dev
```

### 系统要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11 (64-bit) |
| 内存 | ≥ 4GB RAM |
| 磁盘 | ≥ 500MB 可用空间 |
| 网络 | 需要访问 AI 提供商 API |

## 3. 首次配置

启动应用后，点击左侧导航栏 **⚙️ 设置**。

### 3.1 配置 AI 图片生成提供商

**内置提供商：**

| 提供商 | 模型 | 获取 API Key |
|--------|------|-------------|
| DALL-E 3 | dall-e-3 | https://platform.openai.com/api-keys |
| 通义万相 | wanx-v1 | https://dashscope.console.aliyun.com/apiKey |

**添加步骤：**

1. 点击 **"添加提供商"** 按钮
2. 从下拉菜单选择预设提供商，或选择"自定义"
3. 填入 **API Key**
4. （可选）修改端点 URL 或模型名称
5. 点击 **"保存"**

**自定义提供商：**

如果你使用其他兼容 OpenAI 格式的 API：

1. 选择"自定义"
2. 填入 **端点 URL**（如 `https://your-api.com/v1/images/generations`）
3. 填入 **API Key**
4. 填入 **模型名称**
5. 点击 **"保存"**

### 3.2 配置电商平台凭据

以拼多多为例：

1. 切换到 **"平台凭据"** 标签
2. 点击 **"添加凭据"**
3. 填入拼多多开放平台的 `Client ID` 和 `Client Secret`
4. 点击 **"保存"**
5. 在拼多多后台完成 OAuth 授权

## 4. 单商品工作流

### 4.1 生成商品图片

1. 点击左侧 **🎨 AI 生成**
2. 选择 AI 提供商
3. 选择 Prompt 模板（服装/数码/家居/食品/通用等 10 个场景）
4. 填写商品主体描述（如"白色陶瓷马克杯，简约风格"）
5. 点击 **"生成"**
6. 等待 AI 生成图片（通常 10-30 秒）
7. 在图片网格中预览结果

### 4.2 合规检查

生成后自动执行合规检查：

- ✅ 图片尺寸 ≥ 800×800
- ✅ 文件格式为 PNG/JPG/WebP
- ✅ 文件大小 ≤ 5MB
- ✅ 无水印/边框检测

不合规的图片会标记警告。

### 4.3 保存为商品草稿

1. 选择满意的图片
2. 点击 **"保存到草稿"**
3. 填写商品名称、价格、库存
4. 系统自动保存草稿（2 秒防抖）

### 4.4 发布到平台

1. 点击左侧 **📤 发布**
2. 系统自动恢复上次草稿
3. 选择目标平台和凭据
4. 检查商品信息
5. 点击 **"发布"**
6. 等待平台返回结果（通常 5-15 秒）

## 5. 批量工作流

### 5.1 准备 Excel/CSV 文件

创建表格，包含以下列：

| 商品名称 | 商品描述 | 价格 | 库存 |
|----------|----------|------|------|
| 白色陶瓷马克杯 | 白色陶瓷马克杯，简约风格，纯白背景，电商产品摄影 | 29.9 | 200 |
| 不锈钢保温杯 | 不锈钢保温杯，银色，磨砂质感，白色背景 | 59.9 | 100 |

- 支持 `.xlsx`、`.xls`、`.csv` 格式
- 最多支持 500 条商品

### 5.2 导入并创建任务

1. 点击左侧 **📋 批量任务**
2. 点击 **"导入商品清单"**
3. 选择目标平台（拼多多）
4. 选择平台凭据
5. 选择 AI 提供商
6. 设置并发数（1-10，建议 3-5）
7. 拖拽或点击上传 Excel/CSV 文件

### 5.3 批量生成

1. 在任务列表中找到刚导入的任务
2. 点击 **"开始生成"**
3. 观察进度条实时刷新
4. 生成完成后状态变为"待确认"

### 5.4 批量确认

1. 点击 **"批量确认"** — 一键确认所有已生成的条目
2. 确认后的条目进入"已确认"状态

### 5.5 批量发布

1. 点击 **"开始发布"**
2. 逐条发布到电商平台（每条间隔 1 秒防限流）
3. 发布完成后查看结果统计

### 5.6 失败重试

如果部分条目生成或发布失败：

1. 点击 **"重试生成失败"** 或 **"重试发布失败"**
2. 系统自动处理失败条目

## 6. 商品管理与草稿

### 6.1 查看商品列表

点击左侧 **📦 商品管理**，查看所有商品的：

- 商品名称、价格、库存
- 当前状态（草稿 / 待发布 / 已发布 / 失败）
- 关联图片
- 创建/更新时间

### 6.2 编辑商品

- 点击商品行右侧 **"编辑"** 按钮
- 修改商品信息后保存

### 6.3 删除商品

- 点击 **"删除"** 按钮
- 确认后删除商品及其关联图片

### 6.4 草稿自动保存

在发布页面编辑商品信息时，系统每 2 秒自动保存一次草稿。

- 关闭页面后重新打开，自动恢复上次草稿
- 也可手动点击 **"保存草稿"** 或 **"清除草稿"**

## 7. 操作日志与数据备份

### 7.1 操作日志

点击左侧 **📊 操作日志**，查看所有操作记录：

- AI 图片生成记录（提供商、耗时、状态）
- 商品发布记录（平台、商品 ID、状态）
- 支持按平台和状态过滤

### 7.2 发布历史

点击 **"发布历史"** 标签，查看各平台的发布详情。

### 7.3 数据备份

点击左侧 **💾 数据管理**：

**导出数据：**
1. 点击 **"导出数据"**
2. 选择导出范围（全部 / 仅商品 / 仅图片 / 仅日志）
3. 保存到本地文件

**导入数据：**
1. 点击 **"导入数据"**
2. 选择之前导出的文件
3. 确认后恢复数据

## 8. 常见问题排查

### AI 图片生成失败

| 现象 | 原因 | 解决 |
|------|------|------|
| 401 错误 | API Key 无效或过期 | 检查设置中的 API Key 是否正确 |
| 429 错误 | 触发限流 | 等待 1-2 分钟后重试，或降低并发数 |
| 超时 | 网络问题或 API 拥堵 | 检查网络连接，重试 |
| 图片不合规 | 尺寸/格式不符合要求 | 调整 Prompt，使用合规检查功能 |

### 平台发布失败

| 现象 | 原因 | 解决 |
|------|------|------|
| OAuth 失败 | 平台凭据过期 | 重新授权 |
| 发布被拒 | 商品信息不完整或违规 | 检查商品标题、图片、类目 |
| 网络错误 | 服务器无法访问平台 API | 检查网络代理设置 |

### 批量任务异常

| 现象 | 原因 | 解决 |
|------|------|------|
| 导入失败 | Excel 格式不正确 | 确保包含"商品名称"和"商品描述"列 |
| 部分条目失败 | 单条数据异常 | 使用"重试失败"功能 |
| 进度卡住 | 并发数过高触发限流 | 降低并发数，重新生成 |

### 启动问题

| 现象 | 原因 | 解决 |
|------|------|------|
| 白屏 | 前端资源未加载 | 重启应用 |
| 数据库错误 | SQLite 文件损坏 | 从备份恢复 |
| 原生模块错误 | better-sqlite3 版本不匹配 | 运行 `npm run electron:rebuild` |

### 获取帮助

- 检查操作日志定位具体错误
- 导出日志文件（应用设置 → 导出日志）
- 提交 GitHub Issues：https://github.com/xiao-fengyu/-/issues
